Detailed Description

This class defines a thread that will run and handle seralization of our SessionInfo classes. Whenever they are created or updated, the SessionList class drops them off on a BlockingQueue that we then read. This allows the serialization to happen outside of the main response thread and maintain a fast response without waiting for our I/O to happen.
