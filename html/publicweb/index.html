Java, C++, or C# - You take your pick

For a Web Application what happens in the browser is only half the story. 100% native JavaScript is great for the browser, but doesn't help when it comes to writing the server half of your Web Application. With Helix, our SDK provides 3 different choices to help you build your server - Java, C++, or C#.

The best part is that the Helix SDK generates all of the boilerplate code that you usually write by hand. We specialize in helping you to create your data objects that model your custom data, and then use these to communicate back and forth with your user interface.

Using a MVC model for our framework, we help you to define your data model and write your controller logic that is visualized using the web application.

  • Save time by creating your data model properly
  • Generate your Data Object classes
  • Define your screens and form layouts and map to your data

Where To Run - Choices, Choices Choices

Multiple Support for running either stand-alone or deployed to an existing runtime as a plugin.

  • We Use Java - Where can we deploy?
    1. Run Standalone - Using Embedded Jetty
    2. Deploy as WAR - to your favorite App Server
  • We Use C++ - Where can we deploy?
    1. Run Standalone - Using Embedded Mongoose
    2. Deploy as an Apache plugin
    3. Compile support for Windows (32/64bit)
    4. Compile support for Linux (32/64bit)
    5. Compile support for Mac (32/64bit)
  • We Use .Net - Where can we deploy?
    1. Run Standalone - Using Embedded .Net HTTP Server
    2. Deploy as an IIS handler
